## FAQ

### Is Cheeta open source?
The platform itself is proprietary, but we're building it transparently. Your code and data remain yours, and we support open source through:
- Free tier for OSS projects
- Cheeta Ethical License
- Built-in sponsorships

### Can I migrate from GitHub/GitLab?
Yes! We have migration tools for:
- Repositories (including history)
- Issues and PRs
- Users and teams
- CI/CD pipelines

### Does AI require sending my code to third parties?
For cloud users, yes (OpenAI/Anthropic with encryption).
For self-hosted users, you can:
- Use local LLMs (no external calls)
- Bring your own API keys
- Use our hosted AI (data encrypted)

Your code never trains external models without explicit consent.

### What about vendor lock-in?
We fight lock-in:
- Git is Git (standard protocol)
- Export all your data anytime
- API for everything
- No proprietary formats
- Self-hosted option available

### How is this different from GitHub/GitLab?
- **AI-first design** (not bolted on)
- **Built-in community** (DevFeed)
- **Integrated job board**
- **Fair licensing** (Cheeta Ethical License)
- **One unified platform** (not acquisitions duct-taped together)
- **Sustainable business model** (support open source)
- **Developer happiness first** (not corporate quarterly goals)

### Who's behind Cheeta?
Built by developers frustrated with the current state of DevOps tools. We're funded by revenue, not VC money, so we can focus on making developers happy instead of hitting growth metrics.
